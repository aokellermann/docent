We currently don't offer a hosted version of Docent but plan to soon. Luckily, self-hosting is easy and free. For larger organizations, we provide white-glove hosting services; please [reach out](mailto:kevin@transluce.org) if you're interested.

### 1. Clone the repo and configure `.env`

```bash
git clone https://github.com/TransluceAI/docent.git
cd docent
cp .env.template .env
```

You should now have a `.env` file at the project root. See [here](/get-started/envvars) for details on how to fill it in.

### 2. Start the backend server and frontend UI

Docker Compose is the easiest way to get started, but you may want a manual installation to support faster development loops (e.g., for hot reloading).

<AccordionGroup>
  <Accordion icon="docker" title="Docker Compose (recommended)">

First ensure [Docker Engine](https://docs.docker.com/engine/install/) and [Docker Compose](https://docs.docker.com/compose/install/) are installed. Then run:

<CodeGroup>
```bash As non-root
DOCENT_SERVER_PORT=8889 DOCENT_WEB_PORT=3001 docker compose up --build
```

```bash As root
# Note that `sudo` strips environment variables, so you have to set them *inside* the command.
sudo DOCENT_SERVER_PORT=8889 DOCENT_WEB_PORT=3001 docker compose up --build
```
</CodeGroup>

Cold build + start should take a few minutes. Once finished, you can run
```bash
docker ps
```

to check that the four following containers are running:

  </Accordion>
  <Accordion icon="rectangle-terminal" title="Manual">

If you don't already have Postgres and Redis installed, you can star them using Docker:
```bash
docker compose -f docker-compose-db.yml up --build
```

after which Postgres and Redis will be available at the addresses set in [`.env`](/get-started/envvars). If you'd like to set up your own databases, please visit the official docs for [Postgres](https://www.postgresql.org/download/) and [Redis](https://redis.io/docs/latest/operate/oss_and_stack/install/archive/install-redis/).

Once your databases are up, run:

<CodeGroup>
```bash uv
uv sync
```

```bash pip
pip install -e .
```
</CodeGroup>

to install the relevant server packages, then

<CodeGroup>
```bash Prod
docent server --port 8889 --workers 4
```

```bash Dev (with autoreload)
docent server --port 8889 --reload
```
</CodeGroup>

to run the server, then

<CodeGroup>
```bash Prod
docent web --build --port 3001
```

```bash Dev (with autoreload)
docent web --port 3001
```
</CodeGroup>

to run the frontend.

  </Accordion>
</AccordionGroup>

Finally, check that you can access the Docent UI at [`http://localhost:3001`](http://localhost:3001).

### 3. Install the Python SDK

In order to load transcripts into Docent, you'll need to install the Docent Python SDK locally.

<CodeGroup>
```bash uv
uv sync
```

```bash pip
pip install -e .
```
</CodeGroup>

<Note>
If you used the local installation in [Step 2](#2-start-the-backend-server-and-frontend-ui), you can skip this step, since you've already installed the repo locally.
</Note>

You're all set! Check out some of our [tutorials](/tutorials) to get started.
